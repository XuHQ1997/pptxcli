# pptx_cli 项目概述

## 项目目标

`pptx_cli` 是一个辅助 Agent 制作 PPT 的命令行工具。它的核心目标不是让 Agent 直接操作复杂的 PPT 对象模型，而是将 PPT 制作流程转化为“模板 + 结构化表单 + 生成/预览”流程。

## 核心能力

- 从已有 PPT 中提取可复用的模板候选
- 根据模板与表单数据生成新的 PPT
- 支持局部替换幻灯片内容
- 支持导出指定页面预览图，便于 Agent 校验结果

## 设计原则

- Agent 优先：Agent 主要读写 JSON，而不是直接编辑 PPT 内部结构
- 半自动提取：模板提取采用“结构检测 + 候选标注 + Agent 确认”模式
- 先易后难：第一阶段优先支持 text/image，暂不追求图表、SmartArt、动画等复杂对象
- 模板显式化：模板建议采用 `template.pptx + manifest.json` 的形式保存
- 可验证：每一步尽量有中间产物，便于预览、校验和调试

## 建议工作流

1. 对 `existing.pptx` 运行模板检查命令，生成候选标注图和候选 JSON
2. Agent 根据标注图和 JSON 决定哪些页面可作为模板、哪些对象是字段
3. 工具根据确认结果生成模板包
4. Agent 填写表单 JSON
5. 工具根据模板包和表单生成新的 PPT
6. 工具导出预览图供 Agent 复核

## MVP 范围

第一阶段建议只覆盖以下能力：

- 提取页面中的文本框、图片和基础位置信息
- 生成候选标注图和候选 JSON
- 生成模板 manifest
- 根据表单填充 text/image 字段并输出新 PPT
- 将 PPT 指定页导出为预览图

## 当前结论

项目整体可行。核心挑战不在 PPT 文件读写本身，而在如何以稳定、低歧义的方式辅助 Agent 从现有 PPT 中识别模板。
