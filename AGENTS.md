# pptx_cli 项目概述

## 项目目标

`pptx_cli` 是一个辅助 Agent 制作 PPT 的命令行工具。它的核心目标不是让 Agent 直接操作复杂的 PPT 对象模型，而是将 PPT 制作流程转化为“模板 / 表单 / 生成 / 预览”的结构化流程。

## 核心能力

- 从已有 PPT 中提取可复用的模板候选
- 以可视化标注图和结构化 JSON 辅助 Agent 选择模板字段
- 根据模板与表单数据生成新的 PPT
- 支持将 Agent 编写的 SVG 插入到模板字段位置或任意位置
- 支持局部替换幻灯片内容
- 支持导出指定页面预览图，便于 Agent 校验结果
- 以单会话后台服务复用 PPT 加载、解析与编辑现场

## 设计原则

- Agent 优先：Agent 主要读写 JSON，而不是直接编辑 PPT 内部结构
- 会话优先：通过 `init -> 多轮操作 -> finish` 模式复用上下文，减少重复参数和重复解析
- 半自动提取：模板提取采用“结构检测 + 候选标注 + Agent 确认”模式
- 先易后难：第一阶段优先支持 text/image，后续扩展到 SVG 等更复杂的图形内容
- 模板显式化：模板建议采用 `template.pptx + manifest.json` 的形式保存
- 可验证：每一步尽量有中间产物，便于预览、校验和调试
- 兼容回退：对 SVG 这类能力预留降级策略，避免不同 PPT 渲染环境带来的兼容性问题

## 建议工作流

1. 执行 `pptxcli init --origin_file existing.pptx` 启动后台 server，加载并解析当前 PPT
2. 对指定 slide 运行模板检查命令，生成候选标注图和候选 JSON
3. Agent 根据标注图和 JSON 决定哪些页面可作为模板、哪些对象是字段
4. 工具根据确认结果生成模板包
5. Agent 填写表单 JSON，必要时为指定字段或指定位置生成 SVG 内容
6. 工具根据模板包和表单生成新的 PPT，并在需要时继续局部修改
7. 工具导出预览图供 Agent 复核
8. 执行 `pptxcli finish` 结束会话并关闭 server

## MVP 范围

第一阶段建议只覆盖以下能力：

- 提取页面中的文本框、图片和基础位置信息
- 生成候选标注图和候选 JSON
- 提供单会话后台服务，复用 PPT 解析与编辑现场
- 生成模板 manifest
- 根据表单填充 text/image 字段并输出新 PPT
- 将 PPT 指定页导出为预览图

## 当前结论

项目整体可行。核心挑战不在 PPT 文件读写本身，而在如何以稳定、低歧义的方式辅助 Agent 从现有 PPT 中识别模板。当前推荐路线是“视觉标注 + JSON 决策”的混合交互方案，并以单会话 server 作为多轮创作与模板提取的运行基础。SVG 插入能力也具备实现空间，但需要在原生矢量插入与兼容性回退之间做好工程设计。
