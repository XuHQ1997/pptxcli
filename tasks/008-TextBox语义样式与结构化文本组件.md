# 任务 008：TextBox 语义样式与结构化文本组件

## 目标

引入 TextBox 组件抽象，用统一的语义类别和结构配置表达 PPT 中常见的文本视觉模块，降低模板提取和内容生成时对细粒度文本样式的依赖。

## 主要工作

- 定义 TextBox 的语义类别，如强调、正常、淡化等，并为每类内置基础样式
- 设计样式参数的可调范围，首版仅开放颜色等少量安全属性，避免 Agent 直接控制过多排版细节
- 支持 TextBox 的结构化配置，如是否带标题、是否带标题前图标、是否带正文区域等
- 明确 TextBox 与普通文本节点、Container、模板字段之间的关系
- 设计模板提取阶段的标注方式，使 Agent 只需识别 TextBox 类型和颜色信息
- 设计 TextBox 到具体 PPT 文本框、图标、背景块等元素的落地规则
- 处理不同语义类别在字号、字重、留白、层次关系上的统一约束

## 交付物

- TextBox 组件模型定义
- TextBox 配置协议或 JSON schema
- 内置样式类别与颜色映射约定
- 覆盖带标题、带图标、带正文等场景的示例输入输出

## 验收标准

- Agent 可通过结构化 JSON 声明 TextBox 的语义类别与结构
- 工具可根据 TextBox 配置稳定生成统一风格的文本模块
- 模板提取时无需识别大量文本样式细节，只需标注类别和颜色等关键信息
- TextBox 可与 Container 组合使用，支撑常见信息卡片和说明块场景
